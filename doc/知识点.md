
# DDP是什么

DDP 是 DistributedDataParallel 的缩写，是 PyTorch 内置的分布式训练工具，核心作用是实现多 GPU 数据并行训练，让多个 GPU 协同工作，提升训练速度和处理能力。

---

# 动态学习率

什么是param_groups：优化器初始化时，会将模型的所有参数（model.parameters()）默认归为 1 个参数组；但实际使用中，也可手动为不同参数（如模型主干、偏置项、权重衰减不同的参数）设置多个参数组（比如给权重和偏置设置不同学习率）。

---

# 混合精度训练

混合精度训练是将 FP16（半精度） 和 FP32（单精度） 结合使用的训练方式：
模型大部分计算（如前向 / 反向传播）用 FP16：占用显存少（仅 FP32 的 1/2）、计算速度快（GPU 对 FP16 支持更优）；
关键参数（如梯度、权重更新）用 FP32：避免 FP16 精度不足导致的训练不稳定、梯度下溢。

---

# 梯度累积的核心目的
梯度累积（args.accumulation_steps）是为了 用小 batch_size 模拟大 batch_size：
比如脚本默认batch_size=32、accumulation_steps=8，相当于用 8 个 32 样本的小 batch，模拟一个 256 样本的大 batch（32×8=256）；
好处：解决 GPU 显存不足的问题，让大 batch 训练可实现（大 batch 能让训练更稳定、收敛更快）。

梯度累积的核心操作是「多步梯度叠加后再更新参数」，对应到损失计算，必须做 “除法抵消”，否则会导致损失被 “放大”：
正常单 batch 训练（无梯度累积）：1 个 batch 的损失 → 反向传播 → 梯度更新，损失值直接对应 1 个 batch 的预测误差；
梯度累积训练（8 步为例）：8 个小 batch 的损失会连续叠加（每步反向传播后梯度不清空，累积到第 8 步再更新）。
如果不除以args.accumulation_steps（8），会出现两个严重问题：
损失被放大 8 倍：总损失 = 8 个小 batch 损失之和，模型会误以为 “预测误差极大”，反向传播时梯度会过大，导致训练震荡、不收敛；
梯度与学习率不匹配：脚本中学习率（args.learning_rate）是按 “模拟的大 batch” 设置的，若损失不缩放，梯度会和学习率脱节，无法正常更新参数。

# 训练模式（model.train ()）vs 评估模式（model.eval ()）的区别
模型在训练时会启用两类 “随机化操作”，用于防止过拟合、提升泛化能力，但这些操作在 “保存权重” 时会造成问题：
Dropout（随机失活）：训练时随机 “关闭” 部分神经元，避免模型过度依赖某几个神经元；
BN 层（批量归一化）：训练时会实时计算当前 batch 的均值和方差，用于归一化操作，且会更新自身的 “移动均值”“移动方差” 参数。
而评估模式（model.eval ()）会禁用这两类操作：
Dropout：不关闭任何神经元，所有神经元正常工作；
BN 层：不再计算当前 batch 的均值 / 方差，也不更新自身参数，直接使用训练过程中累积的 “移动均值”“移动方差”。


为什么保存模型时必须切换到评估模式？
选中代码的核心目的是「保存模型权重，用于后续加载复用（如续训、推理）」，若不切换到评估模式，会导致两个严重问题：
权重不稳定，加载后效果异常：
若在训练模式下保存，BN 层会继续更新 “移动均值”“移动方差”（哪怕只是保存权重的瞬间），导致保存的 BN 层参数是 “临时的、不完整的”；
后续加载该权重时，无论是续训还是推理，都会因为 BN 层参数异常，导致模型效果偏差（如 loss 飙升、推理准确率下降）。
权重一致性差：
训练模式下，Dropout 的随机关闭会导致模型输出不稳定，虽然不直接影响权重本身，但保存权重时，若 BN 层处于更新状态，会让每次保存的权重存在细微差异，不利于后续复现和对比。

- 优化器状态：学习率、动量、权重衰减、梯度动量等

# 什么是 MoE 架构？
MoE（Mixture of Experts，混合专家模型）是一种模型并行 + 能力分工的深度学习架构，核心思想是：不训练一个超大的单一模型，而是训练多个「专家模型」（Expert），再通过一个「门控网络」（Gating Network）为不同的输入选择最合适的专家来处理。
专家网络（Expert）：多个独立的子模型（通常是 MLP 层），每个专家擅长处理某类输入 / 任务（比如不同语义、不同领域）
门控网络（Gating）：轻量级的路由模块，接收输入后计算每个专家的权重，选择 Top-K 个专家来处理当前输入
混合输出：把选中专家的输出按门控权重加权求和，得到最终结果

好处：参数量大但计算量小，总参数量可以做到超大（比如千亿 / 万亿），但每个输入只激活少量专家（通常 2-4 个），实际计算量和普通模型相当；

# 什么是 RoPE？
RoPE（Rotary Position Embedding，旋转位置编码）是一种为 Transformer 模型设计的位置编码方式，核心思想是：通过对 token 的特征向量进行旋转操作，将位置信息编码到注意力计算中，解决了传统位置编码（如绝对位置编码）在长文本、外推场景下效果差的问题。


# logits是什么
logits：模型最后一层输出的未经过 softmax 激活的原始预测值
维度通常是 [batch_size, seq_len, vocab_size]，代表每个位置对每个词表 token 的原始预测分数。

# KL散度是什么
KL 散度的核心是：衡量两个概率分布之间的差异程度（也叫 “相对熵”）。

数学公式：
$$
D_{KL}(Q \parallel P) = \sum_{i} P(i) \cdot \log\left(\frac{P(i)}{Q(i)}\right)
$$
说明：Q 是学生模型的概率分布，P 是教师模型的概率分布，KL 散度衡量 “用 Q 拟合 P” 的差异程度。
