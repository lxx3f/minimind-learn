# 本地conda虚拟环境配置遇到的问题
---
问题一：conda 创建环境失败（服务条款未接受）
现象：创建 conda 环境时报错：
```
CondaToSNonInteractiveError: Terms of Service have not been accepted
```
原因： Anaconda 近期新增要求，需要显式接受官方频道的服务条款。
解决方法：
```
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/mainconda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/rconda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/msys2
```
---
问题二：scikit-learn==1.5.1 在 Python 3.13 下编译失败
现象： pip install -r requirements.txt 时 scikit-learn 从源码编译，报错：
```
cc1plus.exe: fatal error: No such file or directory
```
原因： scikit-learn 1.5.1 没有 Python 3.13 的预编译 wheel，需从源码编译；Windows 路径长度超过 MAX_PATH（260字符）限制导致编译失败。
解决方法： 将 conda 环境从 Python 3.13 降级到 Python 3.12：
```
conda deactivateconda env remove --name minimindconda create --name minimind python=3.12conda activate minimind
```

Python 3.12 下 scikit-learn 1.5.1 有预编译 wheel，无需编译，安装成功。

---
问题三：ujson==5.1.0 在 Python 3.12 下编译失败
现象： pip install -r requirements.txt 时 ujson 从源码编译，报错：
```
error: Microsoft Visual C++ 14.0 or greater is required.
```
原因： ujson 5.1.0 没有 Python 3.12 的预编译 wheel，需从源码编译，但系统未安装 MSVC 编译器。
解决方法： 将 requirements.txt 中的 ujson==5.1.0 改为 ujson>=5.1.0，pip 会自动选择有预编译 wheel 的版本（如 5.10.0）：
```
# 修改 requirements.txt 后重新安装
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple
```

# AutoDL云服务器环境配置

问题一：网速，下载项目太慢
解决方法：手动添加dns解析
```
# 临时添加GitHub的国内解析IP（重启容器后失效，需重新执行）
echo "192.30.255.112 github.com" >> /etc/hosts
echo "192.30.255.113 github.com" >> /etc/hosts
```

一串警告
```
FutureWarning: The pynvml package is deprecated. Please install nvidia-ml-py instead. If you did not install pynvml directly, please report this to the maintainers of the package that installed pynvml for you.
  import pynvml  # type: ignore[import]
```
原因：
swanlab 包依赖了 pynvml，而 pynvml 这个包已经被废弃（deprecated）。PyTorch 在初始化 CUDA 时检测到系统安装了 pynvml，于是提示你应该改用它的替代品 nvidia-ml-py。
如何消除警告：
```
pip uninstall pynvml -y
pip install nvidia-ml-py -i https://mirrors.aliyun.com/pypi/simple
```
卸载旧的 pynvml，安装官方推荐的 nvidia-ml-py 即可。不过由于 swanlab 声明依赖 pynvml，重新运行 pip install -r requirements.txt 时可能会再次装回来，忽略这个警告也完全没问题。
